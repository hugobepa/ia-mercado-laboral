# 🚨 Error de Especificación - Spec-Kit No Inicializado Correctamente

El error es claro: **Spec-Kit no encuentra el archivo de especificación** porque faltó un paso en el flujo.

---

## 🔍 Diagnóstico del Problema

| Paso | Estado | Archivo Esperado |
|------|--------|-----------------|
| 1. `/speckit.constitution` | ✅ Parece completado | `.specify/constitution.md` |
| 2. `/speckit.specify` | ❌ **FALTA** | `specs/main/spec.md` |
| 3. `/speckit.clarify` | ⏸️ Bloqueado | - |

**Problema:** Ejecutaste `/speckit.clarify` **antes** de `/speckit.specify`. Spec-Kit necesita la especificación base primero.

---

## ✅ Solución: Ejecutar `/speckit.specify` PRIMERO

### **Opción A: Recomendada (Vía Copilot Chat)**

En tu ventana de **Copilot Chat** en VS Code, ejecuta:

```
/speckit.specify

Crear especificación base para el proyecto "IA y Mercado Laboral".

Requisitos:
- Landing page única con secciones ancla
- Datos de Randstad 2024 + OECD 2023 + Idescat
- Secciones: Hero, Datos España/Cataluña, Provincias, Edad/Género, Sectores, Fuentes
- Gráficas ApexCharts (barras, donut, líneas)
- Tablas responsive (desktop: tabla, mobile: cards)
- Diseño: Paleta Mediterráneo Urbano (terracota #C86405, verde #4A7C59, crema #F5F1E6)
- Tipografía: Manrope (títulos) + Inter (cuerpo)
- Mobile-first, accesibilidad WCAG AA
- Sin footer en móvil, botón "volver arriba"
- Página de fuentes dedicada (/fuentes)

Skills a utilizar:
- astro
- tailwind-css-patterns
- frontend-design
- accessibility-wcag
```

---

### **Opción B: Manual (Crear Archivo)**

Si prefieres crear el archivo manualmente, crea esta estructura:

```powershell
# Crear carpetas
New-Item -Path "specs/main" -ItemType Directory -Force

# Crear archivo spec.md
@'
📋 Especificación Base - IA y Mercado Laboral

## Qué
Landing page que explica el impacto de la IA en el empleo en España y Cataluña.

## Por Qué
Target: gente sin estudios universitarios. Necesitan información clara, coloquial y visual.

## Requisitos
- Hero section con título impactante + CTA
- Datos Randstad (20.4M ocupados España, 3.8M Cataluña)
- Desglose por provincias (Barcelona, Girona, Lleida, Tarragona)
- Estadísticas por edad y género
- Sectores más afectados (Comercio, Hostelería, Tech)
- Gráficas ApexCharts interactivas
- Tablas responsive
- Página de fuentes (/fuentes) con 8 fuentes mínimas

## Diseño
- Paleta: Terracota #C86405, Verde #4A7C59, Crema #F5F1E6
- Tipografía: Manrope + Inter
- Mobile-first
- Bordes asimétricos: rounded-t-xl rounded-b-lg
- Accesibilidad WCAG AA

## Datos
- src/data/randstad-catalunya.json
- src/data/idescat-provincias.json
- src/data/charts-config.json
- src/data/fuentes.json
'@ | Out-File -FilePath "specs/main/spec.md" -Encoding utf8
```

## 📋 Flujo Correcto (No Saltar Pasos)

```
┌─────────────────────────────────────────────────────────────┐
│  1. /speckit.constitution  ✅ (ya hecho)                   │
│     → Establece principios rectores                         │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  2. /speckit.specify  ⏳ (HACER AHORA)                     │
│     → Crea especificación base en specs/main/spec.md       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  3. /speckit.clarify  ⏳ (después)                         │
│     → Aclara dudas antes de planificar                      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  4. /speckit.plan  ⏳                                       │
│     → Plan técnico con stack elegido                        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  5. /speckit.tasks  ⏳                                      │
│     → Lista de tareas accionables                           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  6. /speckit.implement  ⏳                                  │
│     → Ejecuta implementación                                │
└─────────────────────────────────────────────────────────────┘
```
